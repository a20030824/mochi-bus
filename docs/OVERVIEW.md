# Mochi Bus 產品總覽

> *Understand the network first, then catch the bus.*

市面上的公車 app 都在回答「我怎麼去那裡」;Mochi Bus 回答的是「**這座城市的公車是怎麼運作的,而我每天的那班車現在在哪**」。前者是導航,後者是理解——這個專案的所有設計都在服務後者。

兩個入口對應兩種時刻:

- **封面(`/`)**:已知路線的極速確認。從口袋掏出手機,開網頁就是答案。
- **地圖(`/map`)**:未知路網的空間探索。路線不是清單,是畫在城市裡的線。

---

## 功能清單

### 封面 `/` — 通勤看板

- 常用站牌的極簡 ETA 看板:超大字級、一眼可讀;30 秒自動刷新、切回分頁瞬間刷新
- 資料降級鏈,**永遠給答案、不給空白**:
  即時 GPS → 時刻表推估(標「依時刻表推估」)→ 被限流時回放最近即時資料(標「稍早」)→ 末班車後顯示「明日 06:10 發車」
- 多看板管理,刪除有 5 秒 inline 復原
- 首次使用是示範模式:LED 車頭燈箱跑馬燈 + 引導卡,不寫入本機資料
- PWA(可加到主畫面、離線提示頁)+ iPhone 捷徑純文字端點 `/shortcut`

### 設定 `/setup`

- 縣市 → 路線(分類 chips + 即時篩選)→ 方向與站牌 → **同站其他公車建議**(依目前選擇、常搭、ETA 排序,一次勾好整個站牌)
- 進階摺疊區:
  - **用詞小抄**:「約」「稍早」「N 分後發車」這些小字的意思、地圖操作提示,結尾致謝 TDX / OSM / Cloudflare
  - **自備 TDX 憑證(BYOK)**:儲存即驗證、憑證只存在裝置端、執行期失效自動退回共用額度
  - 清除本機資料(常用站牌、封面設定與憑證一併刪除)

### 地圖 `/map`

- 全台 22 縣市,區域 → 縣市 → 路線 → 支線 → 站牌的漸進探索;IP 定位跳縣市(不觸發瀏覽器授權框,離台太遠就放棄猜)
- **全路網模式**:整座城市數百條路線與全部站點一次攤開
- 站牌:點地圖找附近、名稱搜尋(索引前綴 + 子字串)、到站時間(即時名額 + 班表補位)、收藏方向到封面
- **點任一站牌,所有經過它的路線線形同時淡色畫出**——站牌不只是等車點,是路網節點;「這一站能通向城市的哪裡」不用查,直接看見
- 路線:支線並列比較選擇、即時車輛位置(20 秒刷新、依方位角旋轉的小巴士圖標)
- 路線規劃:點圖或打字選起終點、直達 + 一次轉乘、ETA 排序、轉乘銜接鬆緊判斷、上車/轉乘/下車區段高亮、起點與終點可各自重選
- 手機返回鍵逐層退回(單一 history 哨兵)、每個畫面都是可分享的 URL

### 底層

- Cloudflare Workers + Hono;D1 放路網快照(不可變版本),R2 放線形、班表與站牌 bundle
- GitHub Actions 每日同步,縣市按星期分片壓在 D1 免費額度內;內容未變由 hash 檢查跳過
- **公路客運攤入縣市路網**:行經本縣市(依 TDX LocationCityCode)的 THB 路線整條併進該縣市快照,站位靠正規化站名 + 200m 網格與市區站牌自然合併;即時查詢按 RouteUID 前綴自動切 InterCity 端點——苗栗這種市區公車只有 27 個站位、公路客運卻有 733 站的縣市,才真的有路網可看
- 三態班表支援(逐站時刻/起點發車/班距制)+ 支線班表互借(雙北班表缺漏的解法)
- 兩層快取(isolate 記憶體 → Cache API)、429 縣市冷卻 + stale 回放
- BYOK:token 以憑證為 key 快取、429 冷卻按憑證來源隔離、回應快取全站共享

## 小巧思

| 巧思 | 為什麼 |
|---|---|
| 點站牌 = 攤開這一站的可能性 | 到站清單之外,所有經過路線一次淡色畫在地圖上——「我從這裡能去哪」是全路網的微縮版,也是整個探索體驗的原點 |
| 路線顏色用名稱 hash | 同一條路線在清單、預覽、路線頁**永遠同色**,大腦能建立「307=磚紅」的連結;轉乘兩段撞色會自動錯開 |
| 觸控/滑鼠差異化 | 觸控疊透明胖線放大命中、不綁 tooltip;滑鼠保留原生精準 hover——不是 RWD,是**輸入方式的 RWD** |
| 底圖去彩度濾鏡 | 整張 OSM 調成灰米色,路線色永遠是畫面主角 |
| 「17:11 發車」不是「還有 131 分」 | 超過一小時沒人心算得動;深夜再給「明日首班」 |
| 「7 分」不是「7 分鐘」 | 一個字的差距,換來 ETA 永不換行 |
| stale 回放 + 「稍早」標示 | 被限流時給舊資料加註記,而不是整面「暫無資料」 |
| 地圖 pane 六層分層 | 預覽小點壓在淡線上、又墊在互動大點下,數百個元素不打架 |
| 縣市按鈕視覺錯開 | 市被縣包住那幾組(雙北、新竹、嘉義…)只挪視覺位置,不動地理座標 |
| LED 燈箱彩蛋 | 一次性、hover 暫停、reduced-motion 靜態、螢幕閱讀器跳過——存下第一塊看板它就永遠開走了 |
| 文案有體溫 | 「公車不是清單,是城市的骨架」「找到你每天在等的那班車,這一頁就會變成你的」 |
| 防抖與防競態 | 雙擊選點只吃一發、預覽用 request id 擋過期回應、定時器與 visibilitychange 不疊加 |

## 與市面軟體比較

| | **Mochi Bus** | Bus+ / 台北等公車 | Google Maps / Moovit | 官方 5284 / iBus |
|---|---|---|---|---|
| 核心形態 | 地圖為中心的**空間探索** | 搜尋框 + 列表的查詢工具 | 導航引擎 | 官方查詢站 |
| 看常用站牌 | 開網頁 = 答案(0 次點擊) | 開 app → 收藏頁(或 widget) | 要搜尋 | 要搜尋 |
| 全路網一次攤開 | ✅ 幾乎獨有 | ❌ | ❌ | ❌ |
| 路線規劃 | 直達 + 一次轉乘(陽春) | 部分支援 | ✅ 完整多模式 + 步行導引 | ❌ |
| 到站提醒推播 | ❌ | ✅ | ✅ | 部分 |
| 帳號/廣告/追蹤 | **全都沒有** | 有廣告或內購 | 帳號生態 | 無廣告但 UX 老派 |
| 安裝 | 免安裝(PWA) | 要裝 app | 要裝 app | 網頁 |
| 自帶 API 額度 | ✅ BYOK | ❌ | ❌ | — |

**定位的取捨**:導航能力(多段轉乘、步行導引、票價、跨運具)刻意不與 Google Maps 競爭;把力氣全部放在「理解路網」與「日常確認」這兩件市面上沒人好好做的事上。全路網模式、支線並列比較、路線恆定配色、漸進式地理探索,都是為此而生。
